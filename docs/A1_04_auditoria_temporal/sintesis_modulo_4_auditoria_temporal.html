Auditoría temporal del GeoPackage regional de puntos de control
PGBM - Actividad 1: diagnóstico regional de aptitud de fuentes y vacíos de información

Este informe sintetiza los resultados del Módulo 4: auditoría temporal, construido a partir del código src/temporal_audit.py y de las tablas generadas en outputs/tables/04_*.csv. El análisis evalúa la integridad del campo Año, su pertinencia frente al año objetivo 2020 y su relación con los grupos XY identificados en el Módulo 3B.

1. Lectura ejecutiva

El campo Año está técnicamente limpio: no presenta valores nulos, vacíos ni fuera del rango plausible definido. La base cubre 23 años, desde 2000 hasta 2022.

La integridad temporal no implica pertinencia temporal. Para una línea base 2020 estricta, solo 225,478 registros corresponden exactamente a 2020, equivalentes a 7.83 % del total. Si se acepta la ventana 2018–2022, la disponibilidad sube a 1,195,911 registros, equivalentes a 41.51 %.

El 58.49 % de los registros está antes de la ventana prioritaria 2018–2022. Por tanto, la base no debe usarse completa como línea base 2020 sin aplicar un criterio explícito de aptitud temporal.

12. Decisiones metodológicas recomendadas

  1. Usar 2020 como año objetivo operativo.
  2. Usar 2018–2022 como ventana temporal prioritaria flexible.
  3. No descartar automáticamente años anteriores a 2018, pero asignarles menor aptitud temporal.
  4. Separar registros exactos 2020 como subconjunto de alta prioridad.
  5. Identificar grupos XY multitemporales cuyo rango incluye 2020 y verificar si contienen observaciones exactas o cercanas a 2020.
  6. Marcar conflictos temáticos con rango que incluye 2020 como revisión prioritaria antes de usarlos como entrenamiento.
  7. Crear una auditoría fuente-país para detectar inconsistencias nominales o mezclas de fuentes.
  8. Combinar el score temporal con calidad espacial, tipo de grupo XY, fuente, clase temática y confiabilidad integrada.

14. Conclusión

El Módulo 4 confirma que la base tiene un campo temporal íntegro y confiable, pero su estructura temporal es altamente heterogénea. El año dominante es 2018, no 2020; el año 2020 aporta solo 225,478 registros. La ventana 2018–2022 mejora la disponibilidad, pero todavía deja 58.49 % de registros antes de la ventana prioritaria. Por tanto, la línea base 2020 debe construirse mediante una estrategia de selección temporal explícita, no mediante uso indiscriminado de todos los registros.

Regla operativa central:

La aptitud temporal debe evaluarse antes de seleccionar muestras de entrenamiento, validación o prueba. 
Para modelamiento, la partición debe respetar xy_group_id y la selección debe priorizar 2020 o la ventana 2018–2022.
