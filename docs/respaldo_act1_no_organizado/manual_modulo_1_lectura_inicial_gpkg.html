Manual de replicación - Módulo 1

Lectura inicial del GeoPackage
PGBM - Actividad 1: Diagnóstico regional de aptitud de fuentes y vacíos de información

Este manual documenta el procedimiento para ejecutar el Módulo 1 del proyecto: la lectura inicial del archivo GeoPackage regional. El objetivo es reconocer técnicamente el archivo sin modificar, limpiar, depurar ni auditar profundamente los datos.

Este módulo permite confirmar que Python puede acceder al GeoPackage, identificar sus capas, reconocer la capa principal, registrar el CRS, el tipo de geometría, el número de registros, los campos disponibles y una muestra inicial de atributos.

1. Propósito del Módulo 1

El Módulo 1 tiene como objetivo realizar una lectura inicial, controlada y reproducible del GeoPackage principal del proyecto. Esta etapa no busca evaluar calidad, eliminar registros ni aplicar reglas de depuración. Su función es generar una primera caracterización técnica del archivo.

El módulo responde preguntas como:

  • ¿El archivo GeoPackage existe en la ruta esperada?
  • ¿Cuántas capas contiene?
  • ¿Cuál es la capa principal sugerida?
  • ¿Cuántos registros contiene la capa?
  • ¿Cuál es el tipo de geometría?
  • ¿Cuál es el sistema de referencia de coordenadas?
  • ¿Cuántos campos atributivos existen?
  • ¿Python puede leer correctamente una muestra del archivo?

Importante: en este módulo no se debe hacer limpieza, corrección, eliminación de duplicados, reclasificación temática ni modificación de geometrías. Es únicamente una etapa de reconocimiento inicial.

2. Requisitos previos

Antes de ejecutar este módulo, debe haberse completado el Módulo 0. Esto implica que ya existen:

  • la carpeta principal del proyecto;
  • la estructura de carpetas;
  • el entorno Conda pgbm_actividad1;
  • el archivo config/config.yaml;
  • las carpetas data/raw/, outputs/tables/ y outputs/reports/;
  • el repositorio Git inicializado.

Ruta del proyecto:

/media/estb/PGB_disco/PGBM_actividad_1_diagnostico

Ruta esperada del GeoPackage:

/media/estb/PGB_disco/PGBM_actividad_1_diagnostico/data/raw/GeoPackage_puntos_control_mesoamerica.gpkg

3. Activar entorno de trabajo

Ejecutar en terminal:

cd "/media/estb/PGB_disco/PGBM_actividad_1_diagnostico"
conda activate pgbm_actividad1

Verificar ruta actual:

pwd

La salida esperada es:

/media/estb/PGB_disco/PGBM_actividad_1_diagnostico

4. Verificar ubicación del GeoPackage

Ejecutar:

ls -lh data/raw/

Debe aparecer el archivo:

GeoPackage_puntos_control_mesoamerica.gpkg

Si el archivo no aparece, puede localizarse con:

find "/media/estb/PGB_disco" -type f -iname "*.gpkg"

Una vez localizado, se puede copiar a data/raw/ o crear un enlace simbólico.

Copiar el archivo

cp "/ruta/donde/esta/el/archivo.gpkg" "data/raw/GeoPackage_puntos_control_mesoamerica.gpkg"

Crear enlace simbólico

ln -s "/ruta/donde/esta/el/archivo.gpkg" "data/raw/GeoPackage_puntos_control_mesoamerica.gpkg"

5. Script principal del Módulo 1

El script usado para este módulo se guarda en:

src/gpkg_inspection.py

Este script genera las siguientes salidas:

Producto Ruta
Inventario de capas outputs/tables/01_inventario_capas.csv
Resumen de capas outputs/tables/01_resumen_capas.csv
Resumen de campos outputs/tables/01_resumen_campos.csv
Capa principal sugerida outputs/tables/01_capa_principal_sugerida.csv
Muestra de atributos outputs/tables/01_muestra_atributos_capa_principal.csv
Reporte Markdown outputs/reports/01_perfil_inicial_gpkg.md

6. Verificar sintaxis del script

Antes de ejecutar el script, se recomienda verificar que no existan errores de sintaxis:

python -m py_compile src/gpkg_inspection.py

Si el comando no imprime ningún mensaje, el script está sintácticamente correcto.

7. Limpiar resultados previos del Módulo 1

Si se hicieron ejecuciones previas fallidas, conviene borrar salidas anteriores para no mezclar resultados. 

rm -f outputs/tables/01_*.csv
rm -f outputs/reports/01_perfil_inicial_gpkg.md

8. Ejecutar el Módulo 1

Ejecutar:

python src/gpkg_inspection.py

Salida esperada:

Módulo 1 ejecutado correctamente.
GeoPackage: /media/estb/PGB_disco/PGBM_actividad_1_diagnostico/data/raw/GeoPackage_puntos_control_mesoamerica.gpkg
Tamaño: 1.43 GB
Capa principal sugerida: Puntos_control_mesoamerica
Salidas generadas en outputs/tables y outputs/reports.

Nota: no se debe ejecutar el script con /usr/bin/python3, porque ese corresponde al Python del sistema. Debe usarse el comando python una vez activado el entorno Conda pgbm_actividad1.

Listar tablas generadas:

ls -lh outputs/tables/

ls -lh outputs/reports/

cat outputs/reports/01_perfil_inicial_gpkg.md

11. Ajuste recomendado en config.yaml

Una vez identificada la capa principal, se recomienda fijarla explícitamente en el archivo: 

config/config.yaml

Cambiar:

gpkg_layer: null

por:

gpkg_layer: "Puntos_control_mesoamerica"

Esto evita ambigüedades en módulos posteriores y permite que los scripts trabajen directamente sobre la capa principal ya validada.

16. Próximo paso

Una vez cerrado el Módulo 1, el siguiente paso será iniciar el:

Módulo 2: Auditoría estructural y tabular

En el Módulo 2 se evaluarán:

  • campos críticos;
  • tipos de datos;
  • valores nulos;
  • dominios de valores;
  • identificadores;
  • duplicados exactos;
  • consistencia estructural;
  • exportación de una tabla auditada de atributos.

El Módulo 2 ya entrará en una auditoría más profunda, pero manteniendo el principio de no modificar los datos originales. Toda salida deberá guardarse en outputs/tables/, data/interim/ o outputs/reports/.
