Manual de replicación - Módulo 0

Configuración inicial del proyecto PGBM Actividad 1
Diagnóstico regional de aptitud de fuentes y vacíos de información

Este manual documenta cómo replicar la configuración inicial del proyecto PGBM - Actividad 1. Su propósito es dejar preparada una arquitectura reproducible para trabajar con datos geoespaciales en Python, usando Conda, Git, GitHub y una estructura profesional de carpetas.

Este módulo no incluye todavía la lectura ni auditoría del GeoPackage. Su función es preparar la base técnica antes de iniciar el análisis de datos.

1. Propósito del manual

El objetivo del Módulo 0 es crear una estructura de trabajo que permita organizar datos crudos, datos intermedios, datos procesados, scripts, notebooks, reportes, bitácoras y documentación técnica de forma clara y reproducible.

Al finalizar este módulo, cualquier integrante del equipo debería poder abrir el repositorio, crear el entorno Conda, entender la estructura del proyecto y continuar con el Módulo 1.

2. Requisitos previos

Antes de iniciar, se requiere:

  • Sistema operativo Linux Mint o distribución Linux equivalente.
  • Disco duro externo montado correctamente.
  • Conda o Miniconda instalado.
  • Git instalado.
  • Acceso a terminal.
  • Editor de código recomendado: VS Code o VS Codium.

Verificar que Conda está disponible:

conda --version

Verificar que Git está disponible:

git --version

7. Mantener carpetas vacías en Git

Git no guarda carpetas vacías. Por eso se crean archivos .gitkeep. 

touch data/raw/.gitkeep
touch data/interim/.gitkeep
touch data/processed/.gitkeep
touch data/external/.gitkeep

touch outputs/tables/.gitkeep
touch outputs/figures/.gitkeep
touch outputs/maps/.gitkeep
touch outputs/reports/.gitkeep

8. Crear el entorno Conda

El archivo environment.yml define el entorno del proyecto. El entorno recomendado se llama: 

pgbm_actividad1

Se usa Python 3.11 y se prioriza el canal conda-forge, especialmente para instalar correctamente librerías geoespaciales como GDAL, GeoPandas, Pyogrio, Fiona, Rasterio, Shapely y PyProj.

Crear el entorno:

conda env create -f environment.yml

Activar el entorno:

conda activate pgbm_actividad1

Registrar el kernel para Jupyter:

python -m ipykernel install --user --name pgbm_actividad1 --display-name "Python (PGBM Actividad 1)"

Verificar que el entorno funciona:

python - <<'EOF'
import sys
import geopandas as gpd
import pandas as pd
import pyogrio
import duckdb
import shapely

print("Python:", sys.version)
print("GeoPandas:", gpd.__version__)
print("Pandas:", pd.__version__)
print("Pyogrio:", pyogrio.__version__)
print("DuckDB:", duckdb.__version__)
print("Shapely:", shapely.__version__)
print("Entorno PGBM Actividad 1 funcionando correctamente.")
EOF

9. Archivos principales creados

9.1 README.md

Describe el propósito del repositorio, la ruta local, los principios de trabajo y el estado de avance por módulos.

9.2 environment.yml

Define las librerías necesarias para reproducir el entorno de trabajo.

9.3 .gitignore

Evita subir al repositorio archivos pesados o sensibles, como GeoPackages, shapefiles, rásteres, archivos Parquet grandes, carpetas de datos, salidas pesadas y archivos temporales.

9.4 config/config.yaml

Define rutas del proyecto, campos esperados, nombres de variables y parámetros generales.

9.5 config/scoring_aptitud.yaml

Define criterios preliminares para el scoring multicriterio de aptitud.

9.6 config/campos_esperados.yaml

Lista los campos esperados en la geodatabase.

9.7 config/clases_homologadas.yaml

Contiene la clasificación temática homologada preliminar.

9.8 logs/decisiones_metodologicas.csv

Registra decisiones técnicas y metodológicas tomadas durante el proyecto.

9.9 docs/

Contiene documentación metodológica, protocolo de auditoría, diccionario de datos y registro de cambios.

10. Inicializar Git

Inicializar repositorio:

git init
git branch -M main

Agregar archivos principales:

git add README.md environment.yml .gitignore pyproject.toml
git add config src docs logs
git add outputs/tables/.gitkeep outputs/figures/.gitkeep outputs/reports/.gitkeep

Como las carpetas data/ y outputs/maps/ están ignoradas por .gitignore, los archivos .gitkeep deben agregarse forzadamente: 

git add -f data/raw/.gitkeep
git add -f data/interim/.gitkeep
git add -f data/processed/.gitkeep
git add -f data/external/.gitkeep
git add -f outputs/maps/.gitkeep

Crear primer commit:

git commit -m "Configura estructura inicial reproducible del proyecto PGBM Actividad 1"

Verificar estado:

git status

La salida esperada es:

On branch main
nothing to commit, working tree clean

11. Ignorar archivos temporales adicionales

Si aparece un archivo .Rhistory, debe ignorarse:

echo ".Rhistory" >> .gitignore
git add .gitignore
git commit -m "Ignora historial temporal de R"

*.shp
*.tif
*.parquet
*.zip

El repositorio debe respaldar principalmente el método, los scripts, notebooks, configuraciones y documentación. Los datos pesados deben permanecer en disco externo, servidor o almacenamiento institucional.
